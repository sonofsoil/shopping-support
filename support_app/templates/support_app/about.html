AI Tool Paradigm: Stacking experts to solve complex tasks

We achieved significant progress in content classification, generation, summarization, expansion and Q&A in the field of Generative AI. The emergent Agent pattern where we assign a role to LLM and give it certain tools to perform complex tasks via multi-turn interaction makes Generative AI even more powerful. Agent paradimg makes the LLM reason and act(ReACT) based on the assigned role and tools in its disposal; the complex task is broken into smaller simpler tasks to achieve final outcome.

The Agent pattern is inherently composible as well. It is very much possible to wrap an Agent as a tool and used by another Agent, the Expert tool. That ability gives rise to level of expertise and building higher-order agents for more complex tasks. This is no different from microservice paradigm, where we built high-order API composing lower-order API.

This is a small demo of a Agent functionality, its composibility and reusability. We used OpenAI GPT-3.5-Turbo as our LLM and LangChang library for Agent implementation. We also show how a organizationl tool registry can accelerate GenAI applications.
